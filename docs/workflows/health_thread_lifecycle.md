# Health Thread lifecycle

1. **Create** — user starts a thread or system suggests one from recurring signals.
2. **Collect** — story, timeline, documents, tests, and prior care are attached.
3. **Connect** — system links related signals, with reasons and confirmation.
4. **Investigate** — an Investigation is opened over the thread; theories are evaluated against personal and external evidence; what changed and what is missing is surfaced.
5. **Clarify** — knowns, unknowns, missing data, and pending items are shown.
6. **Prepare** — user creates a visit packet or self-review plan.
7. **Care interaction** — user attends visit, call, triage, referral, or test.
8. **Post-visit capture** — plan, pending items, uncertainty, and corrections are entered.
9. **Close loops** — results, referrals, follow-ups, and symptom check-ins are tracked.
10. **Resolve/monitor/reopen** — thread is closed, monitored, or reopened when new evidence appears.
