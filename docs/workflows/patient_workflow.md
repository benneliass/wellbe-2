# Patient workflow

## 1. Start or update a Health Thread

The user selects: new concern, recurring concern, pending result/referral, or post-visit follow-up.

The system captures:

- what is happening in the user's own words
- what changed from normal
- when it started and how it changed
- daily-life impact
- main fear or question
- prior visits/tests/referrals
- current medications and recent changes
- access/language/caregiver needs

## 2. Build memory

The system links the thread to existing raw context: prior symptom logs, documents, labs, wearable trends, notes, referrals, and past visits.

The user confirms or rejects suggested links.

## 3. Clarify before action

The system shows:

- what is known
- what is patient-reported
- what is clinically documented
- what is pending
- what is missing
- what remains unexplained
- what changed from baseline

## 4. Prepare for care

The user generates a Visit Packet with selected scope and chooses whether to share/export.

## 5. After visit

The user checks whether concerns were addressed, what is pending, who to contact, when to expect results, and what symptoms should trigger care.

## 6. Close or continue

The thread remains active until resolved, explained, monitored, or archived by the user.
