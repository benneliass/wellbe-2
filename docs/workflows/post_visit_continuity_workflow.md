# Post-visit continuity workflow

The post-visit workflow is the biggest v2 upgrade over the v1 investigation flow.

## Questions after a visit

- Which concerns were addressed?
- Which concerns were deferred?
- What explanation was given?
- What remains uncertain?
- Were tests ordered?
- Are results pending?
- Was a referral placed?
- Who should the user contact and when?
- What changes should trigger urgent care?
- Does the user understand the plan?
- Does the user want to correct the memory?

## Outputs

- updated Health Thread state
- pending item ledger entries
- referral lifecycle status
- normal-test safety net when relevant
- follow-up reminders
- correction requests
- updated Visit Packet for next encounter

Guardrail: WellBe does not claim the visit was clinically adequate or inadequate. It helps the user remember what is open, unknown, or pending.
