# Clinician handoff packet

The Visit Packet is a user-controlled, source-linked summary. It is not an EHR and not a clinician command center.

## Default one-page sections

1. Reason for visit in the user's words
2. What changed from normal
3. Timeline of key events
4. Current symptoms and daily impact
5. Tests/results/referrals already done
6. Pending items
7. Normal tests that did not resolve symptoms, if any
8. Missing context checklist
9. User's top questions
10. Source links and confidence labels

## Design rules

- one-page default
- source-linked details available on drill-down/export
- uncertainty explicitly shown
- patient-reported and clinician-documented information separated
- no final diagnosis claim
- user chooses exactly what to share
