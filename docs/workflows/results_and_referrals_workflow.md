# Results and referrals workflow

## Result flow

1. User imports or logs a test order/result.
2. Result is attached to a Health Thread or kept unassigned.
3. System records status: ordered, pending, resulted, abnormal, reviewed, explained, actioned, closed, unknown.
4. If normal and symptoms persist, the normal-test safety net asks what remains unexplained and what follow-up plan exists.
5. If abnormal or unclear, the system avoids interpretation beyond source wording and helps user track who to contact.

## Referral flow

Referral status taxonomy:

- discussed
- requested
- sent
- accepted
- returned for more information
- wrong service
- no pathway available
- advice-only
- scheduled
- attended
- report received
- next step captured
- closed
- unknown

The referral tracker exists because referral placement is not the same as referral completion.
