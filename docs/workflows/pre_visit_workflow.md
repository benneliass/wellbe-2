# Pre-visit workflow

1. User chooses a Health Thread or creates one.
2. Story Memory asks short adaptive questions.
3. System detects linked prior evidence and asks for confirmation.
4. Missing context checklist asks only high-value questions.
5. Safety Engine checks for urgent-language/red-flag patterns and routes to appropriate urgent guidance where needed.
6. Visit Packet is generated:
   - patient words
   - timeline
   - baseline change
   - related prior visits/tests/referrals
   - pending items
   - normal-test context if relevant
   - open questions
   - source links
7. User previews and chooses what to share.

Design constraint: this is not a clinician intake replacement by default. It is a personal preparation and shareable context tool.
