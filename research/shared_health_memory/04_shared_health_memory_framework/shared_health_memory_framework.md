# Shared Health Memory Framework

## Definition

**Shared Health Memory** is a source-linked, patient-clinician second brain that tracks unresolved Health Threads across time, preserves the patient's story, connects clinical signals, and closes follow-up loops under clinician review.

## Main object: Health Thread

A **Health Thread** is a living container for one unresolved or ongoing health concern.

A Health Thread should contain:

- patient story
- timeline
- baseline change
- daily-life impact
- related visits
- tests and results
- referrals
- clinician assessments
- open questions
- pending actions
- owners
- escalation criteria
- source links

## Medical loop

Replace the generic second-brain loop with:

**Capture -> Connect -> Clarify -> Close -> Correct**

- **Capture:** collect patient story and relevant clinical information.
- **Connect:** link scattered signals into Health Threads.
- **Clarify:** show knowns, unknowns, missing items, and unresolved issues.
- **Close:** track pending results, referrals, follow-ups, and handoffs.
- **Correct:** allow patient and clinician to repair missing or wrong memory.

## Core principle

The product should help healthcare remember; it should not act as final diagnostic authority.
