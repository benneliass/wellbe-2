# Shared Health Memory Research Package

**Research topic:** Shared Health Memory: a medical second-brain approach for unresolved health issues  
**Date packaged:** 2026-05-30

This archive converts the researched "second brain in health" concept into structured files. It separates:

- researched facts
- interpretation and synthesis
- product framework
- architecture
- MVP/features
- safety and regulatory risks
- source log

## Core conclusion

The best healthcare-specific second-brain approach is **Shared Health Memory**: a source-linked patient-clinician memory layer organized around unresolved **Health Threads**.

It should help care teams and patients remember what healthcare commonly loses: patient story, unresolved symptoms, prior tests, pending results, referrals, uncertainty, and follow-up obligations.

## What this is not

This is not an autonomous diagnostic engine. It is not a generic personal notebook. It is not a data factory alone.

The recommended framing is: **memory, continuity, correction, and closure under clinician review**.
